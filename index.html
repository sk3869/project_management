Create a small website using a structured AI‑assisted workflow that demonstrates I can control drift, write a stable spec, run small sprints, and ship a reliable final build.

Spec & Constraints

What the spec does

The spec acts as the single source of truth for this project. Once written, it freezes key decisions so the AI cannot silently reinterpret the goal or expand the scope.

Core constraints

  • Single‑page site focused on process, not visuals.
  • Static stack: HTML + CSS only.
  • Content must explain workflow concepts in plain language.
  • Every change must be traceable to a sprint.

How AI “thinks” here

The AI does not remember like a human; it predicts the next token. Without written constraints, it will drift—adding features, changing tone, or rewriting decisions.

By externalizing the goal, spec, and sprint docs, I give the AI a stable frame it must respect in every subsequent step.

Sprints & Tasks

Why sprints?

Sprints break the build into small, testable units. Each sprint has a single goal, a clear scope, and explicit acceptance criteria.

Example sprint

  • Goal: Define and save the project goal.
  • Scope: Text only, no HTML/CSS changes.
  • Tasks: Draft goal, refine, commit to Git.
  • Acceptance: Goal is stable, specific, and saved.

Receipts

Each sprint ends with receipts: concrete evidence that the plan was followed.

Example receipts
$ git status
$ git add .
$ git commit -m "Add Sprint 001: Project Goal"

These receipts make the process auditable and show that progress is real, not just claimed.

Drift & Checkpoints

Understanding drift

Drift happens when the AI slowly moves away from the original goal—adding features, changing style, or forgetting constraints.

The antidote is simple: keep the goal, spec, and sprint docs external, and refer back to them explicitly in every new request.

Checkpoint packet

A checkpoint is a small “save game” summary I can paste into any new chat to restore project state.

Checkpoint snapshot
  • Project: AI‑assisted workflow demo site
  • Current sprint: Structure & content
  • Completed: Goal sprint + basic layout
  • Next: Refine copy, add README, deploy

What “Success” Means

This project is successful if a reader can understand, from this single page, how a structured AI‑assisted workflow works: how a goal is set, how a spec constrains the build, how sprints and tasks are defined, how drift is controlled, and how receipts and checkpoints keep the project stable over time.

The site itself is intentionally small. The real achievement is a repeatable way of thinking that can scale to larger, more complex projects without losing control.
